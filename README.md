## 📖 О проекте

`music_analytics` — это проект, представляющий собой Python-скрипт для анализа истории прослушиваний пользователя. Он считывает данные из JSON-файла, вычисляет ключевые метрики и визуализирует результаты с помощью графиков.

---

## 📂 Структура проекта
```
music_analytics_project/
├── music.json            # Файл с данными о треках (история прослушиваний)
├── music_analytics.py    # Основной Python-скрипт для анализа и визуализации
└── README.md             # Файл документации
```

## ⚙️ Примеры работы

- **Вывод консольного отчёта:**
При запуске скрипта в консоли отображается следующий отчёт:
```
--- Анализ статистики прослушиваний ---
1. Суммарное время прослушивания: 125.50 мин -> 2 ч 5 мин
2. Топ-жанр: Поп (суммарно 220 прослушиваний)
3. Самый прослушиваемый трек: 'Солнечный берег' от Sunny Crew (95 прослушиваний)
```
- **Визуализация данных:**

После вывода текста автоматически открываются два графика:

1. Столбчатая диаграмма "Количество прослушиваний по жанрам", где каждый жанр имеет свой цвет.
2. Горизонтальная диаграмма "Топ-5 самых прослушиваемых треков" с градиентом от светлого к тёмному.

## 💡 Использование

**Шаг 1: Установка зависимостей**

Установите библиотеку для визуализации:
``` bush
pip install matplotlib
```
**Шаг 2: Запуск программы**

Запустите скрипт из терминала в корневой папке проекта:
``` bush
python music_analytics.py
```
## 🧠 Логика анализа

**Алгоритм работы скрипта:**

- Чтение данных из JSON-файла.
- Расчёт метрик: суммарное время, топ-жанр, самый прослушиваемый трек.
- Визуализация результатов с помощью Matplotlib.

**Особенности реализации:**

- Суммарное время прослушивания рассчитывается как `длительность_трека * количество_прослушиваний`.
- Для топ-жанра учитывается суммарное количество прослушиваний всех треков в жанре.
- Визуализация градиента создаётся с помощью цветовой карты (colormap).

## 🛠️ Технологии

- **Python 3.8+**
- Matplotlib: библиотека для построения графиков и диаграмм.
- JSON: формат файла для хранения и чтения данных.
- Стандартная библиотека: json (для работы с файлами), random (для генерации цветов).
